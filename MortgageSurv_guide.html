Overview

Which path is for you?

The key difference: simulation generates event times for you (you provide statistical parameters). With real data, you already have the event times — no parameters needed.

📊 Path A · Simulation

  • You don't have real mortgage data
  • You want to reproduce the paper
  • You want to explore the package first
  • Must specify β and θ parameters
  • Data is synthetic, not real

📁 Path B · Real Data

  • You have a real mortgage CSV
  • Event times already in your data
  • No β or θ parameters needed
  • Data must be in long format
  • One row per borrower per period
Good news: once you have the sim object — from either path — all modelling, plotting and evaluation code is identical. The two paths only diverge in Step 1.

1 Step 1

Get your data

You provide model parameters (β and Weibull θ) and the package generates borrowers, covariates and default times for you.

What are β and θ?
β (betas) — regression coefficients linking covariates to default risk. One row per landmark period.
θ (thetas) — Weibull shape & scale parameters controlling when defaults happen.
R · Simulated data
sim <- simulate_mortgage_data(
  n            = 1000,   # number of borrowers
  n_iterations = 7,      # landmark periods (IMPORTANT: must match landmarks vector later)
  seed         = 1234    # for reproducibility
)

# What you get back (all generated automatically):
# sim$time_matrix        → (n × 7) default/censoring times
# sim$status_matrix      → 1=default, 0=censored, NA=already defaulted
# sim$x1_matrix          → interest rate covariate
# sim$x2_matrix          → CPI inflation
# sim$x3_matrix          → loan-to-income ratio (LTI)
# sim$x4_matrix          → borrower age
# sim$obsdate_matrix     → observation dates
# sim$credit_start_dates → origination dates
⚠️ Remember: length(landmarks) in Step 4 must equal n_iterations. With 7 iterations → use 7 landmark values: c(54, 56, 58, 60, 62, 64, 66)

You already have event times, statuses and covariates. from_real_data() simply converts your data into the matrix format the package expects. No β or θ needed.

Required format — one row per borrower per period:

ColumnTypeExampleDescription
idint/chr1, "LN-42"Unique borrower / loan ID
iterationint1, 2, 3…Landmark period number (starts at 1)
timenumeric62.3Months from origination to event/censoring — already in your data!
status0 / 111 = defaulted, 0 = censored
x1, x2…numeric0.035Your covariates (any names, any number)
obs_dateDate2020-03-15Observation date for this period
credit_startDate2015-01-10Mortgage origination date

R · Real data
# Load your dataset
my_data <- read.csv("my_mortgages.csv")

# Just map your column names — no betas, no thetas, no simulation!
sim <- from_real_data(
  data           = my_data,
  id_col         = "loan_id",
  time_col       = "months_to_event",   # already in your data!
  status_col     = "defaulted",         # 1=default, 0=censored
  covariate_cols = c("interest_rate", "inflation",
                     "lti", "age"),
  date_col       = "obs_date",
  start_date_col = "origination_date",
  iteration_col  = "period"
)

# Validate everything looks correct
validate_mortgage_data(sim)
From here on, the code is 100% identical to the simulation path. The sim object has exactly the same structure either way.

4 Step 4

Dynamic Landmarking model

Fits a separate Cox model at each landmark using LOCF covariates — re-calibrated at every time-point.

⚠️ Key rule: length(landmarks) must equal n_iterations from Step 1. With 7 iterations → 7 landmarks.
R
landmarks <- c(54, 56, 58, 60, 62, 64, 66)  # 7 values for 7 iterations

df_long <- prepare_landmark_data(
  time_matrix        = sim$time_matrix,
  status_matrix      = sim$status_matrix,
  x1_matrix          = sim$x1_matrix,
  x2_matrix          = sim$x2_matrix,
  x3_matrix          = sim$x3_matrix,
  x4_matrix          = sim$x4_matrix,
  obsdate_matrix     = sim$obsdate_matrix,
  credit_start_dates = sim$credit_start_dates,
  landmarks          = landmarks
)

lm_fit <- fit_landmark_model(
  df_long = df_long,
  x_L     = landmarks,
  x_hor   = landmarks + 12
)

6 Step 6 · Simulation only

Pseudo-evaluation metric

Approximates expected default time for a median borrower per period, using the true β and θ. Only makes sense when you know the data-generating process — i.e. with simulated data.

R · Simulation only
ev <- evaluate_landmark_risk(
  matrices = list(sim$x1_matrix, sim$x2_matrix,
                   sim$x3_matrix, sim$x4_matrix),
  betas = matrix(c(
     50, 0.90, 0.80, -0.20,  51, 0.92, 0.81, -0.21,
     49, 0.88, 0.79, -0.19,  52, 0.95, 0.82, -0.22,
     48, 0.89, 0.78, -0.18,   5, 0.90, 0.80, -0.20,
     51, 0.91, 0.81, -0.21
  ), nrow = 7, byrow = TRUE),
  thetas = matrix(c(
    0.5, 22.25, 0.5, 22.49, 0.5, 22.50, 0.5, 22.73,
    0.5, 22.96, 0.5, 23.19, 0.5, 23.41
  ), nrow = 7, byrow = TRUE),
  seed = 42
)
ev$result
6 Step 6 · Real data

Evaluate on real data

With real data you don't know the true β/θ, so the pseudo-evaluation metric doesn't apply. Instead, extract predicted probabilities and assess them against observed outcomes.

Recommended: Use predict_risk_aft() and predict_risk_ph() to get predicted probabilities, then evaluate with AUC, Brier score, or calibration plots.
R · Real data evaluation
risk_aft <- predict_risk_aft(
  fits$weibull_aft, newdata = df_aft, landmarks = landmarks
)

# risk_aft[["60"]] → predicted % default risk at landmark 60 for each borrower
summary(risk_aft[["60"]])
hist(risk_aft[["60"]], main = "AFT predicted risk at landmark 60")
