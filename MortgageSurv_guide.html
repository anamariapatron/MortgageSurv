MortgageSurv
v0.1.0

Survival analysis models for mortgage default prediction. Adapts biostatistical methods to the UK mortgage market — works with simulated or real data.

Author: Ana M Patrón Piñerez · 2025 · MIT License

Overview

MortgageSurv implements Proportional Hazards, Accelerated Failure Time (AFT), and dynamic Landmarking models in the context of mortgage default. The event of interest is mortgage origination (birth) and default is reinterpreted as death, enabling novel application of classical survival frameworks to credit risk.

The package supports two workflows: generating a synthetic UK mortgage portfolio from scratch, or loading a real dataset and mapping your own column names.

Installation

devtools::install("C:/Users/HP/Downloads/MortgageSurv", upgrade = "never")

# From GitHub

library(MortgageSurv)

Step 1 — Get your data

Generate a synthetic UK mortgage portfolio. The package uses a piecewise Weibull Proportional Hazards model to simulate default times from your parameters.

β (betas) — regression coefficients linking covariates to default risk. One row per landmark period.
θ (thetas) — Weibull shape and scale parameters controlling when defaults occur. Default values are pre-calibrated to the UK market — you can use them as-is.
R
sim <- simulate_mortgage_data(
  n            = 1000,   # number of borrowers
  n_iterations = 6,      # landmark periods — must match landmarks vector later
  seed         = 1234
)

# sim$time_matrix        n × 7 matrix of default / censoring times
# sim$status_matrix      1 = default, 0 = censored, NA = already defaulted
# sim$x1_matrix          interest rate covariate
# sim$x2_matrix          CPI inflation
# sim$x3_matrix          loan-to-income ratio (LTI)
# sim$x4_matrix          borrower age
# sim$obsdate_matrix     observation dates
# sim$credit_start_dates origination dates (length n)
Important: length(landmarks) in Step 4 must equal n_iterations. With n_iterations = 6 use landmarks <- c(54, 56, 58, 60, 62, 64). Landmarks must be within the obs_time range of your data (check with summary(df_long$obs_time)).

If you have a real mortgage dataset, use from_real_data() to convert it into the matrix format the package expects. No β or θ parameters needed — your event times are already in your data.

Required format

One row per borrower per landmark period:

ColumnTypeExampleDescription
idint / chr1, "LN-42"Unique borrower or loan ID
iterationint1, 2, 3…Landmark period number, starting at 1
timenumeric62.3Months from origination to event or censoring
status0 / 111 = defaulted, 0 = censored
x1, x2…numeric0.035Covariates (any names, any number)

credit_startDate2015-01-10Mortgage origination date
Not sure about the format? Run View(example_real_data(n = 10, m = 3)) to see a correctly structured example.
R
my_data <- read.csv("my_mortgages.csv")

# Map your column names — no betas, thetas, or simulation required
sim <- from_real_data(
  data           = my_data,
  id_col         = "loan_id",
  time_col       = "months_to_event",
  status_col     = "defaulted",
  covariate_cols = c("interest_rate", "inflation", "lti", "age"),
  date_col       = "obs_date",
  start_date_col = "origination_date",
  iteration_col  = "period"
)

validate_mortgage_data(sim)   # optional sanity check
From this point on, all code is identical to the simulation path. The sim object has exactly the same structure either way.

Step 3 — Static survival models

Fits Cox PH, Weibull PH, and Weibull AFT models simultaneously on a one-row-per-borrower dataset.

R
# Build one-row-per-borrower dataset
df_aft <- prepare_aft_data(
  time_matrix        = sim$time_matrix,
  status_matrix      = sim$status_matrix,
  x1_matrix          = sim$x1_matrix,
  x2_matrix          = sim$x2_matrix,
  x3_matrix          = sim$x3_matrix,
  x4_matrix          = sim$x4_matrix,
  obsdate_matrix     = sim$obsdate_matrix,
  credit_start_dates = sim$credit_start_dates
)

# Fit Cox + Weibull PH + Weibull AFT
fits <- fit_static_models(df_aft)

summary(fits$cox)
summary(fits$weibull_ph)
summary(fits$weibull_aft)

# Cumulative hazard plot: KM + Weibull PH + Weibull AFT
plot_cumhaz_static(fits$weibull_ph, fits$weibull_aft)

Step 4 — Dynamic Landmarking model

Fits a separate Cox model at each landmark using last-observation-carried-forward (LOCF) covariates. The model is re-calibrated at each time-point, making predictions dynamic.

Key constraint: length(landmarks) must equal n_iterations from Step 1. With n_iterations = 6, use exactly 6 landmark values. Landmarks must also fall within the obs_time range of your data.
R
# 6 landmarks for 6 iterations — within obs_time range
landmarks <- c(54, 56, 58, 60, 62, 64)

df_long <- prepare_landmark_data(
  time_matrix        = sim$time_matrix,
  status_matrix      = sim$status_matrix,
  x1_matrix          = sim$x1_matrix,
  x2_matrix          = sim$x2_matrix,
  x3_matrix          = sim$x3_matrix,
  x4_matrix          = sim$x4_matrix,
  obsdate_matrix     = sim$obsdate_matrix,
  credit_start_dates = sim$credit_start_dates,
  landmarks          = landmarks
)

lm_fit <- fit_landmark_model(
  df_long = df_long,
  x_L     = landmarks,
  x_hor   = landmarks + 12   # prediction horizon: 12 months ahead
)

Step 6 — Pseudo-evaluation (simulation only)

Approximates the expected default time for a median borrower at each landmark period using the true simulation parameters β and θ. This is only meaningful when the data-generating process is known.

R
ev <- evaluate_landmark_risk(
  matrices = list(sim$x1_matrix, sim$x2_matrix,
                   sim$x3_matrix, sim$x4_matrix),
  betas = matrix(c(
     50, 0.90, 0.80, -0.20,  51, 0.92, 0.81, -0.21,
     49, 0.88, 0.79, -0.19,  52, 0.95, 0.82, -0.22,
     48, 0.89, 0.78, -0.18,   5, 0.90, 0.80, -0.20,
     51, 0.91, 0.81, -0.21
  ), nrow = 7, byrow = TRUE),
  thetas = matrix(c(
    0.5, 22.25, 0.5, 22.49, 0.5, 22.50, 0.5, 22.73,
    0.5, 22.96, 0.5, 23.19, 0.5, 23.41
  ), nrow = 7, byrow = TRUE),
  seed = 42
)

ev$result   # expected default time per period for median borrower

Step 6 — Evaluation on real data

With real data the true parameters β and θ are unknown, so the pseudo-evaluation metric does not apply. Instead, extract predicted probabilities and assess against observed outcomes.

Use predict_risk_aft() and predict_risk_ph() to obtain predicted probabilities, then evaluate with standard discrimination and calibration metrics (AUC, Brier score, calibration plots).
R
risk_aft <- predict_risk_aft(
  fits$weibull_aft,
  newdata   = df_aft,
  landmarks = landmarks
)

# risk_aft[["60"]] → predicted % default risk at landmark 60, one value per borrower
summary(risk_aft[["60"]])
hist(risk_aft[["60"]], main = "Predicted risk at landmark month 60",
     xlab = "Predicted default probability (%)")
